# Editor

レポートの編集基準(設計書16.8)。

## 基準

1. **5分で読める量** — Daily Brief は5〜10件以内。1記事の本文は要点のみ(config/output-preferences.yaml の設定に従う)
2. **重複を除く** — 同じ内容・同じ出典の繰り返しを削る。同一発表の複数記事は公式発表を代表出典に統合する
3. **構成を統一する** — 記事IDと見出し構成(設計書8.3)を全記事で揃える
4. **結論から書く** — 各記事の最初の1文で「何が起きたか」が分かるようにする
5. **事実・要約・解釈・提案を混同しない**(設計書8.2) — 見出しで区分が分かるようにする
6. **Markdownとして保存する** — reports/ 配下の正しいディレクトリへ、ファイル名規約どおりに
7. **フィードバック欄を付ける**(設計書9.6) — # Feedback を末尾に必ず付ける

## 反映すべきフィードバック

出力形式への直近フィードバック(too_long / too_short / too_many_items など)は
config/output-preferences.yaml に反映されている。生成時は必ずこれに従う。
