# Source Validator

出典検証(設計書16.9・20章)。**このチェックを通らないレポートは完成扱いにしない。**

## チェックリスト(設計書20章)

レポート本文に対して以下を確認し、結果を本人に報告する:

- [ ] すべての外部情報に出典リンクがある
- [ ] 出典タイトルが記載されている
- [ ] 発行元が記載されている
- [ ] 公開日が記載されている
- [ ] 一次情報と二次情報を区別している(種別表記)
- [ ] 出典の内容と要約が一致している(summary・原文にない主張を書いていないか)
- [ ] Claudeの解釈を事実として記載していない(「確認された事実」に解釈が混ざっていないか)
- [ ] 同じ出典が不要に重複していない
- [ ] レポート末尾に Sources 一覧がある
- [ ] 各記事に記事IDが付いている(DB-/WB-/MR-/DD-)
- [ ] フィードバック欄(# Feedback)がある
- [ ] 過去のフィードバックが適切に反映されている(直近の summaries と矛盾しないか)

リンク切れ確認は `scripts/validate_links.py` が週次CIで行う。レポート生成時に疑わしいURLがあれば WebFetch で確認する。

## 不合格時

- 不足項目を列挙し、修正してから再チェックする
- 出典が確認できない記述は、削除するか「推測」と明示する(設計書3.4)
- 信頼性が低い情報源(設計書8.6)を使う場合は、その旨をレポートに明示する
