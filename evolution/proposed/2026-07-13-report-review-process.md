## 背景 / 課題

- 現状、レポートの品質保証は [source-validator](../../.claude/skills/source-validator/SKILL.md) の**自己チェックリスト1段のみ**。
- これは「出典表記の形式」中心で、**独立した第三者による内容の裏取り(事実確認)・校閲**が存在しない。
- 執筆者(Claude)自身がチェックするため、思い込み・出典の読み違い・二次情報の孫引きを見逃すリスクがある。

## 提案する変更

レポート保存前の工程を「執筆 → 自己チェック → **独立レビュー** → 修正 → 保存」に拡張する。

### 追加する役割(2種のレビュワー)

1. **事実レビュワー(裏取り担当)**
   - 執筆者とは別のサブエージェントとして起動する(独立性の担保)。
   - 各主要主張について、出典に本当にその記述があるか、数値・日付・固有名が一致するかを照合する。
   - 二次情報を一次情報のように書いていないか、解釈を事実として書いていないかを指摘する。
   - 疑わしい主張は WebFetch で原文を再確認する。
   - 出力:主張ごとに 確認済み / 要修正 / 出典不足 の判定と、根拠URL。

2. **校閲(可読性・一貫性担当)**※任意・軽量
   - 誤字脱字、見出し構成、[output-preferences](../../config/output-preferences.yaml) との整合(段落長・箇条書き・空行等)を確認する。

### レビューのしきい値(重すぎない運用)

- **Deep Dive / Weekly / Monthly**:事実レビュワーを**必須**にする(主張が多く影響が大きい)。
- **Daily Brief**:掲載3件以下かつ全て一次情報の日は自己チェックのみ可。外部数値・二次情報を含む主張が1つでもあれば事実レビュワーを起動。
- レビュー結果は各レポートの `# Feedback` の上に `# Review` 節として残す(何を確認し、何を直したかの透明性)。

### 変更対象ファイル

- 各生成スキル(morning-brief / deep-dive / weekly-brief / monthly-self-review)の手順に「独立レビュー」ステップを追記。
- source-validator は「形式チェック」として維持し、事実レビューはその後段に置く。
- 必要なら専用スキル `report-reviewer` を新設する。

## 想定コスト / リスク

- サブエージェント起動によりトークン・時間が増える(特に Deep Dive)。→ しきい値で Daily の軽い日は免除。
- レビュワーも同じ Claude なので独立性は限定的。→ プロンプトで「執筆者の主張を疑い、反証を試みる」姿勢を明示して緩和。
