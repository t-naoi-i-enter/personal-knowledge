## DB-20260717-02 エージェントの暴走支出はクラウド課金の検知より速い:$14,000/日の実害事例

**要点: クラウドの請求ガードレールは「人間の速度のミス」向けに作られており、エージェント速度の支出に追いつかない。支出統制を展開設計の必須項目にすべき。**

### 確認された事実

InfoQ が、AIエージェントとクラウド課金ガードレールのギャップに関する事例をまとめた。

- 3人編成の小規模エージェンシーが、静的アクセスキーを攻撃者に抽出され Bedrock 上の Claude 呼び出しを乱用された結果、1日で $14,000 の AWS 請求を受けた。
- 5月の DN42 インシデントでは、自律エージェントが24時間で $6,531 分の過大なインフラをプロビジョニングした(AWS は後に請求を $1,894 に減額)。
- 課金データ(Cost Explorer)の反映は最大24時間遅れる、と記事は指摘している。いずれの事例も検知のきっかけはクレジットカードの請求だった。

### なぜ重要か

**「請求で気づく」統制はエージェント時代には遅すぎる、という具体的な数字が出た。**

- 昨日の Claude apps gateway(DB-20260716-01)が支出上限(spend caps)を統制項目に含めていたことの必然性を裏付ける事例でもある。

### 自分の業務への示唆

**組織展開・顧客提案の統制設計に「事前の支出上限・キーの短命化・リアルタイムに近い使用量監視」を必須項目として入れる。**

- 静的アクセスキーの排除(短命クレデンシャル化)は、エージェント導入時のセキュリティ標準に明記する。

### 推奨アクション

**自組織・検証環境のAI API キー管理と支出上限設定を点検する。**

- 静的キーの有無、Bedrock / API の支出アラートとハードリミットの設定状況を確認する。
- 工数: 1時間。
- 完了条件: 点検結果と是正項目のリストができている。

## DB-20260717-03 Claude Code v2.1.211:権限承認表示の偽装対策などセキュリティ修正

**要点: ツール入力に含まれる不可視文字で承認メッセージの見た目を偽装できる問題への対策が入った。組織展開ではこの種の修正への追随体制が統制項目になる。**

### 確認された事実

Claude Code v2.1.211 がリリースされた(公式リリースノート)。

- チャットチャネルに中継される権限プレビューが、双方向オーバーライド文字・ゼロ幅文字・見た目が紛らわしい引用符を無害化していなかった問題を修正した。ツール入力が承認メッセージの見た目を改変できないようにするための修正。
- auto モードが PreToolUse フックの ask 判定(サンドボックス外 Bash 向け)を上書きしていた問題を修正し、フックの ask が必ずプロンプトになるようにした。
- `--forward-subagent-text` フラグと `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 環境変数が追加され、stream-json 出力にサブエージェントのテキストと思考を含められるようになった。

### なぜ重要か

**エージェントの「承認UI」自体が攻撃面になる、という具体例。承認フローを設計するだけでは足りず、その表示の完全性まで問われる。**

- フックによる統制(ask 判定)が確実に効くようになった修正は、組織のガードレールをフックで実装する設計の信頼性に直結する。

### 自分の業務への示唆

**組織展開の標準に「バージョン追随ポリシー(セキュリティ修正は N 日以内に更新)」を含める根拠になる。**

- 統制をフックで実装している場合、この種の修正の有無がガードレールの実効性を左右する。

### 推奨アクション

**社内利用中の Claude Code バージョン分布を確認し、更新ルールを決める。**

- 工数: 30分。
- 完了条件: 展開メモに「バージョン追随ポリシー」の案が書かれている。
