# Values

価値観・志の履歴。上書きせず追記する(設計書6.4・22.1)。相反する価値観も矛盾として排除せず、同時に存在するものとして管理する(設計書22.4)。

更新は本人承認型(approval_required: value_statement)。

```yaml
values_history:
  - date: 2026-06
    theme: career
    statement: 技術と経営の両輪で組織を牽引したい

  - date: 2026-07
    theme: purpose
    statement: 社会課題の解決の一端を担いたい
```

## 同時に存在する意志(決めつけない)

- 社長になりたい
- 社会課題を解決したい
- 今の会社を大切にしたい
- 新しい環境にも挑戦したい
